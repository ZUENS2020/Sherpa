# 发布门禁

本文档定义了从开发验证晋级到生产发布前，至少需要满足的检查项。

## 必需检查

### 后端

- 对本次修改涉及的模块执行 `python -m py_compile`
- 运行与变更的工作流、API 或质量逻辑相关的 pytest 子集

### 前端

- 执行前端构建
- 运行受影响包要求的测试或 lint 步骤

### 部署就绪性

- dev 部署使用了预期镜像标签
- worker 镜像与配置保持一致
- 前端消费的 API 字段仍存在且已写入文档
- 如果工作流或 API 行为发生变化，文档已同步更新

## 推荐 smoke test

在把 `dev` 提升到 `main` 之前，至少验证：

- 一个 parser 相关仓库，如 `fmt` 或 `libyaml`
- 一个构建更敏感的仓库，如 `zlib` 或 `libarchive`

目标是同时覆盖规划、生成、构建、运行、崩溃分析链路。

## 出现以下情况时拒绝发布

- 阶段作业在没有实质状态变化的情况下循环
- 覆盖率改进持续 replan，却没有策略变化
- 种子生成退化成源码吞入或大量噪声
- worker 回退到了不受支持的执行假设
- 无法从集群运行态证明部署镜像版本

## 需要保留的证据

每个候选发布版本应保留：

- PR 链接
- 验证命令与结果摘要
- 一个成功的 dev 任务 ID
- 如果变更涉及工作流路由或部署逻辑，则附带回滚说明
