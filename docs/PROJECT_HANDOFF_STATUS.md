## 当前高层状态

Sherpa 当前已经具备：

- 分阶段的 Kubernetes 执行模型
- 以规划、生成、构建、运行、覆盖率改进、崩溃分诊、harness 修复与复现为核心的工作流
- 前后端动态任务与系统视图
- 具备种子质量感知能力的覆盖率改进链路

## 当前仍然存在的工程风险

- 目标深度选择仍可能偏向浅层包装函数
- 若生成轮次过弱，脚手架一致性仍可能被破坏
- 种子质量依旧是实际覆盖率增长的关键决定因素
- 崩溃分类质量仍依赖复现链路与日志质量
